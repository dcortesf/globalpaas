E2 - Gestión de permisos

Cuando el equipo de Global PaaS crea un proyecto, se asignan permisos de administrador al usuario peticionario. Este usuario se encargará de conceder permisos al resto del equipo.

¿Qué necesitamos?

  1. 2 minutos
  2. Credenciales de OpenShift
  3. Proyecto OpenShift

¿De dónde saco las credenciales y el proyecto?

Si aún no tienes acceso a OpenShift, puedes utilizar el OnBoarding de GreenLight. Tienes más información aquí

Paso 1:

Nos conectamos a OpenShift: oc login <fqdn>

./oc login https://api.boae.paas.gsnetcloud.corp:8443

Paso 2: Asignamos los permisos

oc policy add-role-to-user edit <user_name> -n <project_name>

oc policy add-role-to-user edit xIS11103 -n MiProyecto

¿Qué tipo de permisos puedo asignar?

Puedes asignar los siguientes:

  1. view para usuarios de solo lectura
  2. edit para usuarios que pueden subir y desplegar elementos en el proyecto
  3. admin para usuarios que pueden conceder permisos a otros usuarios

Paso 3: Comprobación de permisos

Podemos revisar los permisos asignados mediante:

./oc describe policyBindings :default -n <project_name>

Paso 4: Eliminación de permisos

Podemos eliminar permisos de la siguiente forma:

oc policy remove-role-from-user view n000001 -n myproject
