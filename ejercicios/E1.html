E1 - Despliegue con plantilla

El ejercicio básico de OpenShift es la creación de una aplicación en un proyecto, haciendo uso de una plantilla predeterminada.

¿Qué necesitamos?

  1. 5 minutos
  2. Credenciales de OpenShift
  3. Proyecto OpenShift
  4. Nombre de la plantilla instalada en el proyecto

¿De dónde saco las credenciales y el proyecto?

Si aún no tienes acceso a OpenShift, puedes utilizar el OnBoarding de GreenLight. Tienes más información aquí

Paso 1:

Nos conectamos a OpenShift: oc login <fqdn>

./oc login https://api.boae.paas.gsnetcloud.corp:8443

Paso 2: Selección de proyecto

Seleccionamos el proyecto sobre el que trabajaremos: oc project <projectName>

./oc project MiFormacion

Paso 3: Creación de aplicación

Creamos la aplicación usando la plantilla proporcionada por el equipo de PaaS: oc new-app --name=<app_name> --template=<template_name> --param=<param_list> --labels=app_name= -n

./oc new-app --name=daniapp --template=javase --param=APP_NAME=daniapp,ARTIFACT_URL= 'http://registry.lvtc.gsnet.corp:8081/service/local/repositories/releases/content/com/produban/spring/spring-boot-app/14.0/spring-boot-app-14.0.jar' --labels=app_name=daniapp

Paso 4: Consultamos el estado

Si el despliegue ha sido correcto, podemos:

  • Consultar el estado de la aplicacion: ./oc status | grep MiAplicacion
  • Consultar los logs del contenedor de ejecución:
    1. oc get pods
    2. oc logs MiPod
